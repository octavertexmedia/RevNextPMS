Inventory Management

Manage room availability, blocks, and restrictions across all channels.

Inventory Management is crucial for preventing overbookings and ensuring accurate availability across all your connected OTA platforms. This guide covers everything you need to know.

Understanding Inventory

Inventory in RevNext Channel Manager represents the number of rooms available for booking on a specific date:

Inventory Components

Total Rooms

The total number of rooms of this type at your property

Available Rooms

Rooms available for booking (Total − Occupied − Blocked)

Blocked Rooms

Rooms temporarily unavailable (maintenance, events, etc.)

Occupied Rooms

Rooms with confirmed reservations

Setting Up Inventory

Initial Inventory Setup

  1. Navigate to Inventory in your dashboard
  2. Select your property and room type
  3. Choose a date range (recommended: next 365 days)
  4. Set the total number of rooms for each date
  5. Configure any blocked rooms for maintenance or events
  6. Save the inventory

Bulk Inventory Updates

You can update inventory for multiple dates at once:

  1. Select a date range in the inventory view
  2. Use the bulk update feature
  3. Set the same inventory value for all selected dates
  4. Or use patterns (e.g., different values for weekdays vs weekends)
  5. Apply the changes

Blocking Rooms

Block rooms when they're unavailable due to maintenance, events, or other reasons:

When to Block Rooms

  • Maintenance: Rooms under renovation or repair
  • Events: Rooms reserved for special events or groups
  • Seasonal Closures: Property closed during off-season
  • Overbooking Protection: Reserve rooms for direct bookings

How to Block Rooms

  1. Go to the Inventory page
  2. Select the property and room type
  3. Choose the date(s) to block
  4. Enter the number of rooms to block
  5. Add a reason/note (optional, for your records)
  6. Save the block

Tip: Blocked rooms are automatically excluded from availability calculations and won't be shown as available on any connected OTA platforms.

Setting Restrictions

Restrictions help you control booking behavior:

Types of Restrictions

Minimum Stay

Require guests to book a minimum number of nights

Maximum Stay

Limit the maximum number of nights per booking

Closed to Arrival

Prevent check-ins on specific dates

Closed to Departure

Prevent check-outs on specific dates

Real-Time Synchronization

Inventory changes are automatically synced to all connected OTA platforms:

Automatic Sync

  • Inventory updates sync every 5-15 minutes (configurable)
  • Immediate sync for critical changes (blocks, restrictions)
  • Automatic adjustment when reservations are made
  • Real-time updates across all connected platforms

Benefit: Real-time sync prevents overbookings and ensures accurate availability across all channels, improving guest satisfaction and reducing manual work.

Best Practices

  • Set up inventory for at least 365 days in advance
  • Review and update inventory regularly
  • Block rooms immediately when maintenance is scheduled
  • Use restrictions to optimize revenue (e.g., minimum stay during peak season)
  • Monitor occupancy rates to adjust inventory as needed
  • Keep an eye on sync logs to ensure updates are successful
  • Set up alerts for low inventory or sync failures
