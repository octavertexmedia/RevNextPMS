Integration Process

Step 1: Get API Credentials

Before connecting, you'll need API credentials from the OTA platform:

  1. Log in to your OTA partner portal (e.g., Booking.com Extranet)
  2. Navigate to the API/Integration section
  3. Request API access or credentials
  4. Copy your API key, secret, and any other required credentials
  5. Note the API endpoint URL if provided

Note: API access approval may take 24-48 hours for some platforms. Contact the OTA's support team if you need assistance.

Step 2: Add Integration

  1. Go to Integrations in your dashboard
  2. Click "Add New Integration"
  3. Select the OTA platform from the list
  4. Enter your API credentials:
    • API Key
    • API Secret
    • Property ID (if required)
    • API Endpoint URL
  5. Test the connection
  6. Save the integration

Step 3: Map Room Types and Rates

After connecting, map your internal room types and rate plans to the OTA's format:

  1. Go to the integration details page
  2. Click "Map Room Types"
  3. Match each of your room types to the OTA's room categories
  4. Map your rate plans to the OTA's rate codes
  5. Verify the mappings are correct
  6. Save the mappings

Step 4: Configure Synchronization

Set up automatic synchronization settings:

  • Sync Frequency: How often to sync (recommended: every 5-15 minutes)
  • Sync Types: Availability, rates, reservations, or all
  • Rate Update Rules: When to push rate changes
  • Inventory Sync: Real-time or scheduled
  • Reservation Pull: Automatic or manual

Real-Time Synchronization

RevNext Channel Manager provides real-time synchronization to prevent overbookings and rate discrepancies:

What Gets Synced

Availability (ARI)

  • Room availability
  • Blocked dates
  • Inventory updates

Rates

  • Base rates
  • Seasonal pricing
  • Promotional rates

Reservations

  • New bookings
  • Modifications
  • Cancellations

Restrictions

  • Minimum stay
  • Maximum stay
  • Closed dates

Troubleshooting

Common Issues

Connection Failed

  • Verify API credentials are correct
  • Check if API access is enabled on the OTA platform
  • Ensure your IP address is whitelisted (if required)

Sync Errors

  • Check sync logs for error details
  • Verify room type and rate mappings
  • Ensure inventory is set up correctly

Rate Mismatches

  • Verify rate plan mappings
  • Check currency settings
  • Review rate update rules

Best Practices

  • Test integrations in a sandbox environment first (if available)
  • Start with one OTA platform and expand gradually
  • Monitor sync logs regularly for any issues
  • Keep API credentials secure and rotate them periodically
  • Set up alerts for sync failures
  • Regularly verify that rates and availability match across platforms
  • Use consistent naming conventions for room types across all OTAs
